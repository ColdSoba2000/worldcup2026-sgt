## 13. Tooling/env quirks

- football-data.org's `score.fullTime` field is **inflated** for `PENALTY_SHOOTOUT`-duration matches: `fullTime = regularTime + penalties` (elementwise). Always use `regularTime` for the true score on those; `fullTime` is only correct for `REGULAR` and `EXTRA_TIME` duration matches.
- The `gh` CLI's default OAuth token lacks the `workflow` scope by default in this environment — creating/editing files under `.github/workflows/` via `git push` fails with "refusing to allow an OAuth App to..." until the user runs `gh auth refresh -h github.com -s workflow` themselves (the agent can't grant this — it requires a browser confirmation). Editing the workflow file through GitHub's web UI is an easy way around this without needing the scope at all.
- The agent's sandbox blocks writing a real API token/secret to *any* file or command line, even a scratch temp file — by design. Setting a `gh secret` must be done by the user themselves (paste into a hidden prompt or the web UI), never by the agent.
- Playwright's `page.route()` network mocking did not take effect in this session's browser tool (real remote data kept coming through) — don't rely on it for future mock-based testing in this environment without re-verifying it actually intercepts first.
- `git rebase` was needed repeatedly this session-arc because the (now-disabled) GitHub Action was committing `data.json` updates to `main` every 5 minutes in parallel with manual pushes — always `git pull --rebase origin main` before pushing if the Action is ever re-enabled.
